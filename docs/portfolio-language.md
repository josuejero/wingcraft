# Portfolio Language

Use only claims that can be reproduced by the generated reports in this repository.

## Current Resume Bullet

Built Wingcraft, a React/TypeScript incident-triage console backed by modular parser packages, seeded incident fixtures, Docker-based ops-lab scenarios, and a GitHub Pages deployment workflow. Added deterministic parser validation that verifies label, priority, and escalation behavior across all seeded incidents.

## After Metrics Generation

Added a CI-generated metrics layer covering parser regression pass rate, seeded incident coverage, escalation handling, source/config scope, build status, bundle size, and deployment readiness through machine-readable reports and GitHub Actions summaries.

## After Full Quality Gates

Expanded quality gates with Vitest coverage, Playwright smoke tests, Lighthouse CI budgets, high-severity dependency audit checks, CodeQL, Dependabot, and OpenSSF Scorecard so parser correctness, UI behavior, demo performance, and repository health are visible to reviewers.

## Avoid These Claims

- Do not claim production incident accuracy.
- Do not claim enterprise DORA performance.
- Do not claim zero vulnerabilities unless the current audit and security tools prove it.
- Do not claim independent parser benchmark results unless a separate benchmark dataset exists.
- Do not imply open-source reuse rights while no license is declared.
